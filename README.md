# ابزار تحلیل لاگ (Log Analyzer)

یک ابزار خط فرمان (CLI) نوشته شده با پایتون برای تحلیل فایل‌های حجیم `access.log` سرورهای وب. این ابزار بدون بارگذاری کل فایل در حافظه، لاگ‌ها را خط به خط پردازش می‌کند تا برای فایل‌های بسیار بزرگ هم کارایی داشته باشد.

## ویژگی‌ها
- **پردازش بهینه:** خواندن فایل به صورت Line-by-Line برای جلوگیری از پر شدن حافظه (RAM).
- **مقاومت در برابر خطاهای فایل:** پارس کردن امن داده‌ها و نادیده گرفتن خطوط خراب و کثیف (Dirty Data) بدون کرش کردن برنامه.
- **گزارش‌دهی:** استخراج تعداد کل درخواست‌ها، IPهای یکتا، نرخ خطا (۴xx و ۵xx) و توزیع زمانی ترافیک.
- **انعطاف‌پذیری CLI:** استفاده از ماژول استاندارد `argparse` برای دریافت مسیر فایل و تعداد آدرس‌های پرترافیک.

## پیش‌نیازها و اجرا
این ابزار نیازی به هیچ کتابخانه جانبی (Third-party) ندارد و فقط با ماژول‌های استاندارد پایتون نوشته شده است. 

برای اجرای برنامه، ترمینال را باز کرده و دستور زیر را وارد کنید:
`python main.py مسیر_فایل_لاگ`

**مثال اجرای استاندارد:**
`python main.py sample_logs/access.log`

**مثال با تعیین تعداد خروجی‌های برتر (مثلاً ۱۵ آدرس پرترافیک):**
`python main.py sample_logs/access.log --top 15`

## چالش‌ها و تصمیمات فنی
در روند توسعه‌ی این ابزار، تصمیمات فنی زیر برای حل چالش‌ها گرفته شد:

۱. **مدیریت حافظه (Memory Management):** برای فایل‌های چند گیگابایتی، استفاده از متدهایی که کل فایل را در حافظه می‌ریزند باعث کرش شدن سیستم می‌شود. به همین دلیل از خواندن خط به خط (`for line in file`) استفاده شد.
۲. **شمارش IPهای یکتا:** در این پروژه از ساختار داده `set` استفاده شده است. اما می‌دانیم که در محیط‌های واقعی با ترافیک‌های میلیونی، به دلیل مصرف بالای RAM توسط `set`، بهتر است از الگوریتم‌های تقریبی و بهینه مثل **HyperLogLog** استفاده شود.
۳. **پارس کردن لاگ و داده‌های کثیف:** از `Regex` استفاده شد و ساختار آن به گونه‌ای تنظیم شد که در صورت وجود نام‌کاربری به جای خط‌تیره در لاگ‌ها (فرمت `%u %l`)، داده‌ها به اشتباه به عنوان `invalid` شناخته نشوند.
